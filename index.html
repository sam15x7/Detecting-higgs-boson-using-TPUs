What is this?

From Collision Events
to Classification

The Higgs boson is the quantum of the Higgs field — the fundamental mechanism that gives all matter mass. Discovered at CERN in 2012, detecting it in new collision data is a needle-in-haystack problem requiring ML at scale. This project tackles it with deep neural networks accelerated on Google TPUs.

⚡

TPU Optimized

Full distributed training on Google Cloud TPU v3 pods with XLA compilation for up to 3× speedup over equivalent GPU setups.

🧠

Multiple Architectures

DenseNet, ResNet-style skip connections, and custom deep MLPs — all benchmarked and tunable via config.

📡

Physics-Aware Metrics

Evaluates with AMS (Approximate Median Significance) — the metric used in the original ATLAS Higgs challenge, not just accuracy.

Model Design

Architecture & Pipeline

Raw 28-dimensional kinematic features flow through a normalization → embedding → deep network stack, trained end-to-end with TPU-distributed gradient accumulation.

01

Raw Features

28 kinematic properties from the ATLAS detector per collision event

02

Preprocessing

StandardScaler normalization + missing value imputation in data_loader.py

03

Embedding Layer

512-dim dense projection with BatchNorm and GELU activation

04

Deep Network

Residual MLP: [512→256→128→64] with skip connections and dropout

05

Classifier Head

64→1 sigmoid for binary classification (signal / background)

Network Layers

The HiggsClassifier in model.py is a fully-configurable residual MLP. Hidden dimensions are passed at init time, enabling quick architecture search:

TPU Training Strategy

The tpu_utils.py module handles all TPU lifecycle — device detection, strategy scoping, and shard management. Training uses tf.distribute.TPUStrategy for transparent multi-core data parallelism.

Key decisions baked into the training loop (train.py):

  • Binary cross-entropy loss with label smoothing 0.1
  • Adam optimizer, lr=1e-3, cosine decay schedule
  • Batch size 2048 per TPU core (16384 global)
  • XLA-compiled @tf.function train steps
  • tf.data pipeline with prefetch + cache for I/O-free training

Quickstart

How to Run

Three paths to run this project: Google Colab (zero setup), Cloud TPU VM, or local GPU fallback.

1

Clone the Repository

Get the project source from GitHub. The zip also contains the full source under Detecting-the-Higgs-Boson-With-TPUs-*.zip.

2

Download the Dataset

Fetch the HIGGS dataset from the UCI Machine Learning Repository (~2.6 GB compressed). Place the CSV in data/.

3

Install Dependencies

All requirements are pinned in requirements.txt. TensorFlow 2.x with TPU support is the primary dependency.

4

Configure TPU (optional)

Set environment variables for Cloud TPU, or let tpu_utils.py fall back to GPU/CPU automatically.

5

Train & Evaluate

Run train.py then evaluate.py for full AMS scoring. Use the notebook for exploration and visualization.

bash — clone & setup 
# Clone the repo
git clone https://github.com/sam15x7/\
  Detecting-higgs-boson-using-TPUs
cd Detecting-higgs-boson-using-TPUs

# Install dependencies
pip install -r requirements.txt

# Download HIGGS dataset (UCI)
wget https://archive.ics.uci.edu/ml/\
  machine-learning-databases/00280/HIGGS.csv.gz
gunzip HIGGS.csv.gz
mv HIGGS.csv data/
bash — Cloud TPU 
export TPU_NAME=higgs-tpu
export TPU_ZONE=us-central1-b
export PROJECT_ID=your-project-id

python train.py \
  --tpu_name $TPU_NAME \
  --project_id $PROJECT_ID \
  --epochs 10
python — basic usage 
from src.data_loader import HiggsDataset
from src.model import HiggsClassifier
from src.train import train_on_tpu

# Load and split data
dataset = HiggsDataset(data_path='data/')
train_loader, val_loader = \
  dataset.get_loaders()

# Instantiate model
model = HiggsClassifier(
  input_dim=28,
  hidden_dims=[512, 256, 128]
)

# Train on TPU
trainer = train_on_tpu(
  model, train_loader,
  val_loader, epochs=10
)
